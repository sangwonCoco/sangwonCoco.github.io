명령어 집합 구조와 마이크로 아키텍처

  • 명령어 집합 구조란?
    • 프로세서가 인식하여 실행할 수 있는 기계어 명령 집합
  • 명령어 집합구조는 명령어와 레지스터 및 메모리 상호작용 등에 관한 모든 내용을 포함하고 있으며, CISC나 RISC 등의 아키텍처에 따라 여러 형태로 구성될 수 있다.
  • '마이크로 아키텍처'는 이런 명령어 집합 구조를 조직하고, 프로세서를 운영하기 위한 하드웨어 수준의 구조를 정의하는 개념이다.

마이크로 아키텍처의 성능 향상 기법

  • 마이크로 아키텍처의 성능은 명령어를 얼마나 최소의 시간 내에 실행할 수 있느냐가 중요한 척도가 되어 발전해왔다.
  • 마이크로 아키텍처의 성능 향상 기법은 여러가지가 있지만, 여기선 중요한 3가지만 살펴보도록 하자.
  1. 연산 장치의 고속화
    • 프로세서의 부하에 많은 영향을 미치는 부분이 연산 장치이다.
    • 프로세서에는 연산 처리를 위해 ALU라는 유닛을 두고 있으며, 마이크로 아키텍처는 연산을 고속화하기 위한 다양한 알고리즘을 제공한다.
    • 초기에는 ALU만을 사용했으나, 부동 소수점 등 복잡한 연산을 고속 처리하기 위한 전용 하드웨어인 FPU라는 장치가 만들어졌다.
  2. 캐시 기술
    • 메모리에 프로세서 사이에 SRAM을 두어 자주 사용되는 데이터를 저장해서 메모리 액세스를 최대한 고속으로 처리하는 기술이다.
  3. 병렬 처리 기술
    • 마이크로 아키텍처에서 처리해야 할 명령어를 최대한 동시에 실행하는 방법을 사용하여 처리 시간을 단축 시키는 기술이다.
    • 이는 하드웨어 수준(마이크로 아키텍처)에서 구현된 병렬 처리 기법이며, "명령어 수준 병렬성"이라 부른다.
    • "명령어 수준 병렬성"을 실현하기 위해 파이프라인, 슈퍼스칼라 등의 기법을 적용한다.

병렬 컴퓨팅

  • 병렬 컴퓨팅 이란?
    • 여러 개의 연산을 동시에 병렬적으로 처리하는 방법을 의미한다.
  • 여러 개의 연산을 병렬로 처리하기 위해서는 동시에 처리하는 연산 간에 의존성이 없어야 한다. 그래야 한 개 연산을 수행할 때 동시에 다른 연산도 수행 가능하다.
  • 병렬 처리의 시작점은 동시에 수행할 수 있는 대상(의존성이 없는 명령어)을 찾는 것이다.
  • 이러한 일련의 과정을 '병렬화'라고 한다.

하드웨어 수준의 병렬 처리 기법 종류

  • 하드웨어 수준의 병렬 처리 기법으로는 2가지가 있다.
    1. 비트 수준 병렬 처리 기법
    2. 명령어 수준 병렬 처리 기법

비트 수준 병렬 처리 기법

  • 한 번에 처리되는 word량을 늘리기 위해 버스의 대역폭을 늘리고, 레지스터나 메모리와 같은 장치의 용량을 넓히는 방법을 사용하는 기법
  • word란?
    • CPU에서 2진 데이터를 처리하는 단위

명령어 수준 병렬 처리 기법

  • 다수의 명령어(마이크로 아키텍처의 여산 처리 기본 단위가 되는 명령어)들을 동시에 수행하기 위한 기법
  • 파이프라이닝, 비순차 실행, 데이터 병렬 처리
CPU의 명령어 처리 과정
  • 명령어 수준 병렬 처리 기법의 종류에 대해 알아보기 이전에 "CPU의 명령어 처리 과정"을 살펴본다.
  • CPU는 일반적으로 명령어의 실행을 제어하는 Control Unit과 데이터의 연산 처리를 담당하는 Data Path로 구성되어 있다.
  • 명령어 처리 과정
    1. Fetch: Control Unit은 RAM에 적재된 명령어를 하나씩 가져온다.
    2. Decode: 패치한 명령어를 해석하고, 적당한 마이크로 명령어를 찾아 Datapath로 전달한다.
    3. Memory Access: Datapath는 Control Unit으로부터 명령어를 전달받아, 연산에 필요한 데이터는 RAM이나 Register에 적재한다.
    4. Execution: ALU에서 실제 연산을 수행한다.
    5. Write Back: 연산을 마친 후 연산 결과는 Register에 저장되고, Store 명령을 통해 최종적으로 메모리에 저장된다.
파이프라이닝
  • 명령어를 단계별로 나누어 한 단계씩 순차적(코드의 흐름에 따라 처리)으로 진행함으로써 명령어가 마치 동시에 실행되는 것처럼 하여 처리 성능을 높이는 기법이다.
  • 하지만 아래의 경우 '파이프라인 해저드' 문제 발생
    • 명령어 간 동일한 자원을 사용
    • 하나의 명령어가 앞선 명령어의 연산 결과에 종속되는 경우
    • 조건 분기에 따른 후속 명령 대기
비순차 실행
  • 명령어의 순서와 상관없이 빨리 처리할 수 있는 순서대로 재배열하는 방법을 사용하여 처리 성능을 높이는 기법이다.
  • 예시
    • 현재 실행되고 있는 명령어 A와 나중에 실행될 명령어 B간에 의존성이 없다면, B의 순서를 앞으로 당겨 A와 B를 동시에 실행하는 방식으로 처리하여 속도를 향상시킨다.
  • 핵심
    • 동시에 실행 가능한 명령어를 최대한 많이 찾아내는 것이다.
  • 이를 실현하기 위해 '슈퍼 스칼라'라는 기술을 사용한다.
    • 슈퍼 스칼라
      • 파이프라인 자체를 여러 개 두어 1사이클에 더 많은 명령어를 처리하도록 하는 기술이다.
      • 이때 각 파이프라인에서 동시에 실행되는 명령은 서로 의존성이 없어야 한다.
데이터 병렬 처리
  • 하나의 명령어에 대해서 여러 데이터를 병렬 처리하는 것은 SIMD(Single Instruction Multiple Data Stream) 구조 내에서 실현된다.
  • 반대로 지금까지 살펴본 하나의 명령어에 대하여 하나의 데이터를 처리하는 것은 SISD(Single Instruction Single Data Stream) 구조 내에서 실현된다.
  • SISD 구조 내에서는 길이가 n인 배열의 모든 요소에 대해 "a[i] = b[i] + c[i]"를 처리하려면 n 만큼 연산을 반복해야 한다.
  • 그러나 SIMD 구조 내에서는 이러한 배열 연산을 단 하나의 명령어로 수행가능하다.
  • SIMD 구조의 핵심은 '연속적인 복수의 데이터를 처리하는 전용 레지스터 집합(Vector Register)'과 '연산의 병렬 처리'라고 볼 수 있다.

싱글 코어의 한계와 멀티 코어의 등장

  • 싱글 코어 환경에선 아래와 같은 하드웨어 수준의 병렬 처리 기법을 통해 성능을 향상시켰다.
    • 비트 수준 병렬 처리
    • 명령어 수준 병렬 처리
  • 그러나 클럭 속도의 향상은 전력 소비량, 발열량의 증가로 이어졌고, 또한 하드웨어의 제약으로 인해 명령어를 동시에 처리할 수 있는 명령어 수준 병렬성 또한 한계성을 드러냈다.
  • 이런 상황에서 칩 하나에 다수에 프로세서를 결합한 형태의 멀티 코어 프로세서가 등장하게 된다.
  • 멀티 코어는 다수의 코어에 의한 병렬 처리를 통해 성능 향상을 꾀하기에 싱글 코어에서 나타났던 문제를 해소할 수 있다.
  • 또한 다수의 코어를 사용함으로 처리 능력을 극대화하고 스레드 수준 병렬성을 제공하여 프로그래밍을 통한 성능 향상이 더욱 유연해 지게 되었다.

병렬 컴퓨터와 멀티 코어

  • 병렬 컴퓨터 정의
    • 다수의 프로세서들이 수 많은 연산을 동시에 병렬 처리할 수 있는 형태로 조직된 컴퓨터
  • 결합 구조에 따라 강 결합 구조, 약 결합 구조로 구분된다.
    • 강 결합 구조
      • 다수의 프로세서가 주기억장치 및 IO를 공유하며, 하나의 OS가 모든 프로세서를 관리하는 구조이다.
      • SMP(Symmetric Multi-Processor)라고도 한다.
      • 멀티 코어는 SMP의 한 예시이다.
        • 여러 개의 코어(프로세서)가 하나의 칩에 집적된 방식으로 CMP(Chip Multi Processor)라고도 부르며 일반 PC에 가장 많이 적용되어 있는 병렬 컴퓨터 형태이다.
        • 즉, 멀티 코어를 구성하는 각 코어들은 하나의 칩에 집적되어 있다는 말이다.
    • 약 결합 구조
      • 다수의 프로세서가 독립적인 주기억장치와 IO를 가지고 있으며, 각각의 OS를 지닌 구조이다.
      • 독립적인 PC들이 네트워크를 통해 상호 연결되는 클러스터나 그리드 컴퓨터 등이 이러한 약 결합 구조로 분류될 수 있다.

멀티 프로세싱! 멀티 코어 시대의 병렬 처리 기법

  • 멀티 프로세싱
    • 다수의 프로세서가 장착된 컴퓨터에서 프로세서 간의 협력을 통해 연산을 처리하는 방식을 의미한다.
    • 즉, 멀티 코어에서는 소프트웨어 수준의 멀티 프로세싱을 통해 다수의 코어를 활용하여 병렬 처리하는 방식으로 성능을 향상시킨다.
    • 다만, 제대로 활용하기 위해서는 처리하고자 하는 연산을 각 코어에 효율적으로 배분할 수 있도록 프로그램을 구현해야 한다. -> 병렬 프로그래밍의 중요성이 매우 커짐

멀티 프로세싱은 어떻게 이뤄지는가?

  • 우선 싱글 코어와 병행성(Concurrency, 동시성)에 대해 알아보자.

  • 싱글 코어는 코어가 하나이기에 여러 개의 작업을 병렬적으로 수행할 수 없다!

    • 따라서 싱글 코어에서는 여러 개의 작업을 동시에 수행하는 방법으로 OS 수준에서 멀티 태스킹이라는 기법을 제공한다.
    • 멀티 태스킹
      • 다수 개의 작업들을 작업 내용이나 우선순위에 따라 일정 시간을 배분하고, 배분된 시간만큼 CPU를 번갈아가며 점유할 수 있도록 스케줄링하는 방식이다.
      • 병행성(Concurrency, 동시성) 이라는 특징을 가진다.

  • 멀티 코어는 멀티 태스킹은 물론 각 코어마다 작업을 나누어 배분하여 병렬적인 처리가 가능하다.

    • 병행성(Concurrency, 동시성), 병렬성(Parallelism) 이라는 특징을 가진다.

  • 멀티 프로세싱 기반에서 각 코어로 배분되는 작업은 프로세스 또는 스레드 단위로 처리된다.

    • 프로세스
      • 메모리에 적재되어 현재 OS에 의해 실행 중인 프로그램
      • 프로세스는 또 다시 스레드라고 하는 경령화된 프로세스들로 나눠진다.
      • 스레드를 여러 개 생성하여 동시에 수행하는 기법을 멀티 스레딩이라고 한다.

  • 싱글 코어에서는 스레드들이 멀티 태스킹을 통해 동시적(Concurrency)으로 처리되는 반면, 멀티 코어에서는 스레드가 각 코어로 배분되어 병렬적으로 처리될 수 있다.

  • OS는 이러한 코어의 전반적인 리소스 상황을 모니터링하여 스레드를 최적의 코어에 배분하기 위한 스케줄링 작업을 수행한다.

  • 이렇게 멀티 코어 환경에서는 컴퓨터가 알아서 병렬 처리를 해 주는 것이 아니라, 프로그램을 구현할 때 동시에 수행될 수 있는 대상을 찾아 스레드를 만드는 작업, 스레드 간의 자원 경쟁을 해소하기 위한 동기화 작업 등을 반드시 수행해야 한다.
