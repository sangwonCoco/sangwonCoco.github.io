문제

A transformation sequence from word beginWord to word endWord using a dictionary wordList is a sequence of words such that:

  • The first word in the sequence is beginWord.
  • The last word in the sequence is endWord.
  • Only one letter is different between each adjacent pair of words in the sequence.
  • Every word in the sequence is in wordList.

Given two words, beginWord and endWord, and a dictionary wordList, return the number of words in the shortest transformation sequence from beginWord to endWord, or 0 if no such sequence exists.

Constraints:

  • 1 <= beginWord.length <= 10
  • endWord.length == beginWord.length
  • 1 <= wordList.length <= 5000
  • wordList[i].length == beginWord.length
  • beginWord, endWord, and wordList[i] consist of lowercase English letters.
  • beginWord != endWord
  • All the strings in wordList are unique.


기본 테스트 케이스

Example 1:

1Input: beginWord = "hit", endWord = "cog", wordList = ["hot","dot","dog","lot","log","cog"]
2Output: 5
3Explanation: One shortest transformation is "hit" -> "hot" -> "dot" -> "dog" -> "cog" with 5 words.

Exapmle 2:

1Input: beginWord = "hit", endWord = "cog", wordList = ["hot","dot","dog","lot","log"]
2Output: 0
3Explanation: The endWord "cog" is not in wordList, therefore there is no possible transformation.

핵심 아이디어

  • 기본적으로 두 개의 단어 셋을 사용한다.
    • reached: 특정 레벨로 도달할 때 변환된 단어 SET 이다.
    • wordDict: 아직까지 도달하지 못한 단어 SET 이다.
  • 자세한 설명은 구현 코드 파트의 주석과 후기 파트를 참고한다.


구현 코드

 1class Solution {
 2    public int ladderLength(String beginWord, String endWord, List<String> wordList) {
 3        Set<String> wordSet = new HashSet<>(wordList); // 아직 도달하지 못한 단어 set
 4        Set<String> reachedSet = new HashSet<>();      // 특정 레벨에 도달했을 때 변환된 단어 set
 5        
 6        // 초기화
 7        reachedSet.add(beginWord);
 8        wordSet.remove(beginWord);
 9        int level = 1;
10        
11        // reachedSet에 원소가 있다는 것의 의미: level n에서 level n+1로 변환될 수 있는 단어가 있다.
12        while (!reachedSet.isEmpty()) {
13            Set<String> reachedNextSet = new HashSet<>();
14            /*
15            현재 레벨에 존재하는 reachedSet의 단어들의 각 문자 요소들을 모두 변경해보며 
16            아직 방문하지 않은 단어인지 체크한다. 만약 아직 방문하지 않은 단어라면 다음 레벨에 들어갈
17            reachedSet의 요소로 넣어준다. 이후 변환 가능한 단어가 정답이라면? 정답(현재 레벨+1)을 반환
18            */
19            for (String r : reachedSet) {
20                for (int i = 0; i < r.length(); i++) {  
21                    char[] rarray = r.toCharArray();
22                    for (char c = 'a'; c <= 'z'; c++) {
23                        rarray[i] = c;
24                        String newString = new String(rarray);
25                        if (wordSet.remove(newString)) {     // 만약 아직 방문하지 않은 단어라면
26                            reachedNextSet.add(newString);   // 다음 레벨에 들어갈 reachedSet의 요소로 넣어줌
27                            if (endWord.equals(newString)) { // 만약 정답이라면
28                                return level+1;              // 정답 (가장 짧은 변환 시퀀스 수) 
29                            }
30                        }
31                    }
32                    
33                }
34                
35            }
36            reachedSet = reachedNextSet;
37            level++;
38        }
39        return 0;        
40    }
41}

  • 도저히 못풀겠어서 Disscuss를 보니 대부분의 사람들이 BFS로 풀었었다. 그래서 나도 다시 BFS로 풀어봤다. 그러나 이번에도 끝내 TLE를 해결하지 못했다.
  • 이후 다른 사람 코드를 보고 이해 한 후 결국 풀어내긴 했다...
  • 모든 문제가 그렇듯 한 번 이해하면 정말 쉽지만 그 과정은 결코 쉽지 않다. 나의 경우는 아래 2가지를 이해하는 것이 정말 힘들었다. 이 문제 같은 경우는 Solution 부분이 비공개 되어 있어서 시각 자료를 참조할 수 없었는데, 이해를 돕기 위한 시각 자료 없이 한글로 천천히 읽어봐도 어려운 내용을 못하는 영어로 읽으며 이해하려 했기 때문인 것 같다.
    1. 다음 레벨로 진입할 때 변환 가능한 단어를 찾기 위해 비교하는 방법을 이해하지 못했다.
      • 왜 각 단어의 각 문자를 하나씩 바꿔가면서 비교하는지 이해할 수 없었는데 아래 설명을 보고 이해할 수 있었다. 
        1// 링크: https://leetcode.com/problems/word-ladder/discuss/40704/Java-Solution-using-BFS-with-explanation
2ps: I get TLE at the first two submissions, 
3because when I check if wordDict has any matches with reached set, 
4I use two for loops and determine if any pair of words differ by one. 
5That's a huge slow-down because it'll takes m (size of reached) * n (size of wordDict) * l (length of words) time, 
6while in this solution, it takes 26 * l * m time. So when n is huge, 
7this solution will be (n/26) times faster.
      • 정리하자면 이 사람도 나와 같이 2개의 문자열을 비교하기 위해 이중 루프를 이용했었고 이로 인해 TLE가 발생했었다고 한다.
      • 왜냐하면 현재 레벨에서의 모든 문자 set에서 다음 레벨로 변형될 수 있는 모든 문자 set을 찾기 위해서는 연산 횟수가 n * l * m 이다. 그러나 각 단어의 각 문자를 하나씩 바꿔가면서 비교할 때의 연산 횟수는 26 * l * m 이다.
        • n: wordDict의 크기
        • l: 각 단어의 길이
        • m: 각 레벨에서의 reached의 크기
      • n(wordDict의 크기)이 거대할 때, 후자가 전자보다 n/26 만큼 빠르다.
    2. 각 하위 레벨로 진입할 때 마다 그 길 만의 고유한 path를 유지해야 한다고 생각했다. 즉, 각 레벨로 진입하는 노드마다 그 노드까지 도달하기 위한 고유한 HashSet(path)를 가지고 있어야 한다고 생각했다.
      • 풀다가 말리다 보니 문제에서 요구하는 것이 'shortest transformation sequence'라는 것을 어느 순간부터 놓치고 있었다. 정확히는 알고는 있었지만 이것으로 부터 아래의 전제를 도출할 수 있다는 것까지 연결시키지 못했다.
        • 문제에서 요구하는 것은 'shortest transformation sequence'이다.
        • BFS 방식으로 접근시, 레벨은 순차적으로 증가하며 각 레벨에 진입할 때 마다 변환 가능한 단어들을 모두 찾을 수 있다.
        • 이런 상황에서 모든 path에서 wordDict와 reachedSet을 공유해도 되는 이유는 다음과 같다.
          • level 1, beginWord = "abe", wordDict = {"abc", "abg", "abf"}인 상황에서
          • level 2로 진입 시 reachedSet은 {"abc", "abg", abf"}가 된다
          • 이때 level 2에서 level 3으로 갈 때 "abc"는 "abg", "abf"가 될 수 있다.
          • 이때 level 2, 3 모두에서 "abg", "abf"가 중복 등장한다.
          • 근데 잘 생각해보면 level 2의 "abg"든 level 3의 "abg"든 그 하위에 나올 하위 path의 모습은 똑같다. ("abf"도 마찬가지)
          • 문제에서 요구하는 것은 beginWord에서 endWord까지 가장 짧은 변형 시퀀스를 구하는 것이므로 더 낮은 레벨에서 이미 나온 단어가 더 높은 레벨에서 똑같이 등장하는 경우는 배제해도 된다는 의미이다.
          • 그러므로 아직 지나지 않은 단어 set을 의미하는 wordDict와 현재 레벨에서의 단어 set을 의미하는 reachedSet을 모든 경로에서 공유해도 상관 없다는 결론을 도출 할 수 있다.
